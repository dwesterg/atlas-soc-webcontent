Software development tools and several example designs are already installed on the SD Card that comes with the Atlas-SoC evaluation board. You can begin immediately by using the Atlas-SoC board as a remote development system via SSH or VNC. For serious development we recommend you download and install the appropriate tool chain on a PC or workstation; details are covered in the "Develop" page.

  • Developing on the Atlas-SoC Board (via SSH or VNC)

    There are two methods for accessing the tools and design examples on the Atlas-SoC board. In both cases you will need to connect your USB cable between your PC and the USB port of the Atlas-SoC board.

     

     

    Secure Shell (SSH)

     

    • Launch a host system terminal program
    • Type: ssh root@192.168.7.1

     

    Windows users can download a free version of PuTTY here.

     

     

    Virtual Network Computing (VNC)

     

    • Launch your host VNC client software.
    • Connect to the VNC Server at 192.168.7.1

     

    A free version of RealVNC can be downloaded here.

     

    For Mac OS X (v10.4 and greater)

    • Open Safari
    • In the URL bar type vnc://192.168.7.1

  • General Purpose I/O

    Example Programs

    Each of the examples below demonstrate how to interact with the GPIO framework provided by the Linux environment running on the Atlas-SoC target. Please see the README files in each subdirectory for more information.

     

    raw-gpio

     

    Demonstrates how to programmatically detect the state of the KEY0 push button on the Atlas-SoC board.  The script and one C program simply read the state, output it and exit, the second C program enables the interrupt functionality for the GPIO and waits for the push button press to generate the interrupt that it waits for.

     

    gpio-keys

     

    Demonstrates how to programmatically detect the input events generated by the SW0, SW1, SW2 and SW3 switches on the Atlas-SoC board. The script and one C program simply report the input events as they are detected, the second C program adds a call to the ioctl() function to read the current state of all four switches as each input event is processed.

     

    gpio-leds

     

    Demonstrates how to programmatically control the LEDS on the Atlas-SoC board.

     

     

    Documentation

    The README files containing information on the following topics:

    • How to build  and run the examples
    • How the GPIO controllers are implemented on the Atlas-SoC board
    • Discovering useful run time device tree information maintained by the kernel in procfs
    • How to interact with the Linux GPIO framework

  • Accelerometer

    Example Programs

    Each of the examples below demonstrate how to interact with the Analog Devices ADXL345 Input 3-Axis Digital

    Accelerometer Linux driver which is deployed on the Atlas-SoC software environment. Please see the README files in each subdirectory for more information.

     

    adxl_calibrate.sh

     

    Shows how to calibrate the accelerometer.

     

    tap_detect.c

     

    Demonstrates how to detect the tap events that the accelerometer produces.

     

    watch_adxl.c

  • Fast Fourier Transform Hardware Accelerator

    Example Programs

    There are four example programs for the FFT application. Two use the ARM CPU & NEON co-processor to perform the FFT calculation, and two which use the FFT hardware in the FPGA. For each of the processing variants (CPU & FPGA) there are two data sources used; one where the data is stored on the SD Card, and one where the data is in FPGA RAM. Be sure to review the README files in each sub directory for more information.

     

    neon32_256.c

     

    This program performs the FFT operation using the ARM CPU and NEON co-processor using data stored on the SD Card.

     

    ffftdma_256.c

     

    This program performs the FFT operation using the hardware accelerator and DMAs inside the FPGA using data stored on the SD Card. The input and output data pass through the Acceleration Coherency Port (APC) of the processor's L2 cache.

     

     

    Documentation

    The README files containing information on the following topics:

    • How to build and run the examples
    • Generating sample data sets
    • Plotting the FFT results

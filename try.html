Developing Software

 

Software development tools and several example designs are already installed on the SD Card installed in the Atlas-SoC evaluation board. If you wish to begin immediately you can use the board as a remote development system via SSH or VNC. If you prefer to develop on your PC, you will need to first set up your software environment. Details on downloading, and installing the tools and example designs are listed in the section titled Set Up Your Software Development Environment below.

  • Set Up Your Software Development Environment

    This section contains:

    • Details on the tools required for development (Linux, SoCEDS, DS-5AE)
    • Instructions for downloading, installing, and/or licensing the tools
    • Instructions for downloading and installing the example design files

    • A section for each example design describing

Developing Hardware

 

Hardware development for SoC devices includes configuration of both the processor, and the programmable logic. Either the Quartus II Web Edition, or Subscription Edition software can be used to accomplish both tasks. Please begin by installing the Quartus II software on your personal computer; details are covered in the Set Up Your Development Environment section below..

  • Set Up Your Development Environment

    This section will cover:

    • Installing and licensing Quartus II
    • Downloading (or copying) the hardware projects for the appropriate Quartus II version

  • Processor Configuration

    This section will cover:

    • Overview of processor configuration
    • Configuring the processor using Qsys/Quartus

  • FPGA Design

    This section will cover:

    • Description of the FPGA project (includes a block diagram)
    • Location of the project files
    • Instructions for compiling the design
    • Instructions for generating the handoff files for the software team

     

  • Configuring The FPGA

    This section will cover:

    • Overview of the SoC boot image
    • Process to combine the hardware and software images into a single boot image
    • Overview of multi-boot images (factory default + user image)
    • Process of creating a multi-boot image
